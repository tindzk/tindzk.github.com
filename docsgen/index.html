Introduction

docsgen is a frontend for the Typography markup.

Its purpose is to create well-formatted documents without much effort. Therefore, docsgen offers various features which facilitate styling and structuring texts.

Installation

Requirements

For tplgen to compile, make sure to have these prerequisites installed:
  • Jivai
  • Jivai-Utils
  • tplgen

Checkout

$ git clone git://github.com/docsgen/docslib.git
$ git clone git://github.com/docsgen/docsgen.git

Compile

$ cd docsgen; ./build.sh

Usage

docsgen requires at least one parameter which is the path to the document. To generate this manual, run: 
$ ./docsgen.bin manual.tyo
Additionally, you can specify a second parameter for the base path in which the CSS file (style.css) and the syntax highlighter are stored.

Commands

A document written in Typography is basically a sequence of (nested) commands surrounded by text. Commands are comparable to functions in programming languages. Hence, some commands may even accept options.

A typical command is used like this: .cmd{value}. Parameters are generally optional. They can be specified using square brackets: .cmd[param]{value}.

Meta

meta is a top-level command, i.e. it cannot be used within other blocks. It allows to specify the document's title: 
.meta {
    .title{Document Title}
}

Chapters and Sections

Like meta, chapter is another top-level command. Chapter are automatically added to the table of contents. Sometimes it's desired to subdivide the contents of a chapter. Therefore, chapters may also contain sections (.section{...}). However, the generated table of contents won't cover any links to sections.

Style

The following styles are supported:
  • b (bold)
  • i (italic)
  • class (ClassName)
  • func (main)
  • var (var)
  • macro (macro)
  • term (1 + 2)
  • keyword (docsgen)
  • path (/root)
  • number (8080)
Styles only format the given text, i.e. they don't require any options.

Blocks

docsgen supports blocks. Their purpose is to inline texts. Notes (note) and warnings (warning) are supported:

Note block

Warning block

Lists

A list is initialized using the list command. Within a list, there cannot be other elements than items: 
.list {
    .item{Item 1}
    .item{Item 2}
}

Paragraphs

docsgen does not detect paragraphs automatically. To force a paragraph, the corresponding text needs to be surrounded by a p command.

Syntax highlighting

docsgen supports highlighting of code and shell commands:
  • command: Shell command(s)
  • code: C, C++ code
For example, this 
.command {
    $ ls -lah
}

.code {
    int main(void) {
        return EXIT_SUCCESS;
    }
}
will render to: 
$ ls -lah
int main(void) {
    return EXIT_SUCCESS;
}

Anchors

Anchors are invisible markers which can be set throughout the whole document and jumped to. An anchor needs to be initialized using the anchor command. A link pointing to an anchor can be generated using jump.

Links

The following types of links are supported:
  • mail: E-mail address (user@example.com)
  • url: URL (example.com)
The usage of a link is similar to an anchor jumpers': 
.url[http://www.google.com/]{Google}
.url[john@example.com]{John Smith}

Example

A sample document may look like: 
.meta {
    .title{Sample Document}
}

.chapter{First Chapter} {
    ...
}

.chapter{Second Chapter} {
    .section{Section} {
        ...
    }
}
