podfm

1 Introduction

podfm is an extensible podcatcher for the console. It provides the possibility to write provider-specific plugins. This is especially useful when providers offer additional content such as transcriptions which general-purpose podcatchers usually ignore.

2 Requirements

  • Jivai
  • Jivai-Utils

3 Installation

First make sure that the aforementioned requirements are met. Then checkout and compile the code using: 
$ git clone git://github.com/tindzk/podfm.git
$ cd podfm; ./build.sh

4 Getting Started

podfm can download files locally to disk or to an external device like a media player. Therefore, podfm maintains a cache file keeping track of already downloaded podcasts.

By default, the configuration path points to the current directory. However, this can be configured in config.h. But you can also pass a different path via parameter.

The configuration path must contain a file called subscriptions.yml with the following layout:

providerId1:
    first subscription
    second subscription

providerId2:
    ...

4.1 RFI plugin

As for RFI, the file may look like: 
rfi:
    Podcast_DANSE
    Podcast_SAV
    Podcast_VEN
    journalFrancaisFacile
    rss_apprendre_francais
This layout stores the selected podcasts in one directory, rfi. If this isn't desired, you may also break them up into multiple sections: 
rfi: RFI Group 1
    Podcast_DANSE
    Podcast_SAV

rfi: RFI Group 2
    Podcast_VEN

If you don't label each section differently, the separation won't have any effect.

4.2 RSS plugin

The RSS plugin needs to be used like this: 
rss: ecoute
    url: http://www.ecoute.de/podcast/rss.xml

rss: tagesschau
    url: http://www.tagesschau.de/export/podcast/tagesthemen/
    url: http://www.tagesschau.de/export/podcast/tagesschau/

url is necessary because the configuration parser will strip http from the URL when nothing else is prepended.

4.3 Limiting downloads

Some news podcasts have a repertoire which goes back for several weeks. If you only want to hear the latest episode or e.g. the last five episodes, you can put the option limit: n into the podcast group. This will download the first n episodes and skip the rest. The default value is -1 which means there's no limit. 
rss: tagesschau
    limit: 5
    url: http://www.tagesschau.de/export/podcast/tagesthemen/

4.4 Date duplication in file name

The titles may already contain the date. If this is the case, the downloaded file name will contain the date twice. This can be prevented by using the option date: no.

4.5 Synchronize

To synchronize, just run: 
$ ./podfm.bin /mnt/ipod
Consequently, the cache file as well as the podcasts will be stored in the given path.

5 Providers

Currently these providers are supported:
  • RFI (rfi)
  • RSS (rss)
The RSS plugin only exists to cover those providers which aren't supported by any specific plugin.
